# MVC 패턴




### 모델2 방식

- 모델1 방식의 단점 보완
- 웹 애플리케이션의 각 기능을 분리해서 구현
- 클라이언트 요청 처리
- 응답 처리
- 비즈니스 로직 처리
- 각 기능이 분리되어 모듈화되어 있으므로 모듈별 개발이 가능
- 모듈을 비슷한 프로그램 개발에 사용할 수 있어 코드 재사용성도 높음
- 응용프로그램 확장성 및 이식성이 좋아짐
- 개발 후 서비스 제공 시 유지보수 편리
- 현재 모든 웹 프로그램은 모델2 방식으로 개발



### MVC 패턴

- M : Model (DTO / DAO)
- V : View (JSP 페이지)
- C : Controller
